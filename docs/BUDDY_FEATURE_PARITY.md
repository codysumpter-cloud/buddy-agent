# Buddy Feature Parity

This document tracks the intended feature surface for Buddy Agent across Hermes Agent, Buddy Brain, Omni Buddy, Prismtek Apps, and Knowledge Vault.

## Runtime parity from Hermes Agent

- [ ] Terminal chat runtime
- [ ] Model provider selection
- [ ] Tool registry and toolsets
- [ ] Persistent memory
- [ ] Skill creation and skill execution
- [ ] Scheduled automations
- [ ] Messaging gateway
- [ ] Sandboxed command execution
- [ ] Subagents and delegated workstreams
- [ ] Browser/web tooling

## Buddy Brain integration

- [ ] Operator startup context
- [ ] Council role contracts
- [ ] Runtime runbooks
- [ ] Workspace sync helpers
- [ ] Codex bridge adapter
- [ ] Skill pack import strategy
- [ ] Decision and continuity records

## Omni Buddy integration

## Definition of done

A feature is not considered complete until it has:

1. A documented owner and boundary.
2. Tests or validation commands.
3. Security and secret-handling notes where relevant.
4. Source provenance recorded when adapted from another repo.
